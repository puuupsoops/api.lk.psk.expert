Собственные директории:
\init - содержит данные для инициализации

Структура модуля:

\lib - содержит все классы и методы,
которыми обладает модуль (автоматическое подключение классов, при соблюдении правил). Не обязательна, если не требуется функционал(отсутствуют собственные методы модуля).


\install - информация о модуле,
содержит файлы и папки для установки и удаления модуля
  install\index.php - файл с кодом установки и удаления модуля, содержит основные параметры модуля
  install\version.php - хранит номер верссии модуля (!версия не равна нулю) + дату выпуска последней версии модуля
  install\step.php [+ step1.php ...etc] - не обязательно, пошаговая установка модуля
  install\unstep1.php [+ unstep2.php ...etc] - не обязательно, пошаговое удаление модуля


\install\admin - содержит файлы, при установки переместятся в папку \bitrix\admin (должен содержать код подключения файла)
    admin\menu.php - файл административного меню модуля
    admin\operation_description.php - файл с описанием для управления прав рассширенного доступа
    admin\task_description.php - файл с описанием уровней доступа для operation_description.php

\lang\id_языка (прим: ru - русский) - папка с языковыми файлами с языковыми константами


\install\db + '\mysql'(<---подпапка с именем базы) - содержит sql файлы, используются при установки модуля


\install\components - папка с компонентами модуля (копируются при установки модуля)
    components\папка_с_пространством_имен (прим: psk ,)
        папка_с_пространством_имен\имя_компонента (прим: psk\api.orm.add , psk\api.class, psk\api.orm.getlist ...etc)

include.php - используется для принудительного подключения файла с классом, при инициализации модуля
options.php - присутствует, если есть страница настроек (прим. прав доступа к модулю, см. в Административном меню: Настройки)
default_option.php - [старое ядро]
prolog.php - [старое ядро]
